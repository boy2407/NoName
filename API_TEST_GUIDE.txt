NoName API Test Guide - Swagger Copy/Paste

Base URL:
- https://localhost:<port>

Chuẩn dùng Swagger:
1) Bấm `Authorize` và dán `Bearer <access_token>` cho API cần auth.
2) API `POST/PUT` thì bật `Try it out` và dán JSON bên dưới.
3) API có path/query params thì nhập đúng giá trị mẫu đã ghi.

==================================================
3) PAYMENTS (MoMo)
==================================================

⚠️ IMPORTANT: Duplicate Prevention
- Nếu gọi POST /api/payments/create/{orderId} 2 lần cho cùng 1 order:
  • Lần 1: Tạo Transaction mới, trả về PayUrl ✓
  • Lần 2: Phát hiện pending/success transaction, trả về PayUrl cũ (không gửi lại MoMo) ✓
- Lý do: MoMo API từ chối duplicate orderId để ngăn chặn duplicate payment
- Benefit: Có thể retry an toàn khi network error

POST /api/payments/create/{orderId}
Path param:
- orderId: 100
Query param:
- provider: MoMo
Expected Response (Lần 1):
{
  "statusCode": 200,
  "message": "Payment created successfully",
  "data": "https://test-payment.momo.vn/..."
}
Expected Response (Lần 2 - Same OrderId):
{
  "statusCode": 200,
  "message": "Payment created successfully",
  "data": "https://test-payment.momo.vn/..."  ← Trả về URL cũ
}

GET /api/payments/momo-callback
Query params:
- orderId: 100
- resultCode: 0
- transactionId: MOMO-100
- message: Success

POST /api/payments/momo-ipn
Body JSON:
{
  "orderId": "100",
  "resultCode": "0",
  "transactionId": "MOMO-100",
  "message": "Success"
}

Payment flow test nhanh (copy làm lần lượt) * kết quả mong đợi
==================================================
1) `POST /api/orders` -> lấy `orderId`.
2) `POST /api/payments/create/{orderId}?provider=MoMo`.
3) `GET /api/payments/momo-callback?orderId=<orderId>&resultCode=0&transactionId=MOMO-<orderId>&message=Success`.
4) `POST /api/payments/momo-ipn` body:
{
  "orderId": "<orderId>",
  "resultCode": "0",
  "transactionId": "MOMO-<orderId>",
  "message": "Success"
}
